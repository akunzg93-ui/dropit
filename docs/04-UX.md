# Guía de UX de Dropit

> Documento Oficial
>
> Versión: 1.0
>
> Estado: Oficial
>
> Última actualización: 08/07/2026

---

# Objetivo

Definir una experiencia consistente para todas las pantallas de Dropit.

El usuario nunca debe sentir que cambió de aplicación al navegar entre módulos.

---

# Principios

- Simplicidad antes que decoración.
- Una acción principal por pantalla.
- La información importante debe verse primero.
- Reducir la cantidad de clics.
- Mantener consistencia visual.

---

# Identidad visual

## Colores

Color principal:

- Azul (#2563EB)

Gradientes:

- #2563EB
- #1E40AF

---

# Componentes

## Cards

- Bordes redondeados.
- Sombras suaves.
- Espaciado uniforme.

---

## Botones

Botón principal:

- Azul.
- Ancho completo cuando sea acción principal.

Botón secundario:

- Blanco.
- Borde gris.

Botón de peligro:

- Rojo.

---

## Inputs

Todos los formularios deberán mantener:

- Altura consistente.
- Bordes redondeados.
- Estados de error visibles.

---

# Iconografía

Se utilizará Lucide Icons como librería oficial.

---

# Logo

El logo deberá mantenerse visible en las pantallas principales.

No deberá reducirse innecesariamente.

---

# Terminología

Utilizar siempre:

- Cliente
- Vendedor
- Establecimiento
- Administrador

No utilizar "Comprador" en la interfaz.

---

# Mensajes

Los mensajes deben ser:

- Claros.
- Breves.
- Accionables.

Evitar mensajes técnicos al usuario.

---

# Responsive

Todas las pantallas deberán funcionar correctamente en:

- Móvil.
- Tablet.
- Escritorio.

---

# Consistencia

Cuando se cree una nueva pantalla deberá reutilizar los componentes existentes antes de crear nuevos.
